<!-- Backend API contract — the exact endpoints the frontend calls -->

From backend/app/schemas/schema.py:
- AgentCreate: { name: str, description: str | None, model_id: int | None, prompt_id: int | None }
  (status field intentionally omitted — always DRAFT on creation)
- AgentRead: { id: int, name: str, description: str | None, model_id: int | None, prompt_id: int | None, status: AgentStatus, created_at: datetime, model_name: str | None, prompt_name: str | None }
  (use_enum_values=True means status is lowercase string: "draft", "active", "inactive")
- StatusChangeRequest: { status: AgentStatus, reason: str | None }
- Response[T]: { code: 0, data: T, message: "success" }

From backend/app/api/v1/endpoints.py:
- GET /api/v1/agents/?skip=N&limit=N&status=X → Response[list[AgentRead]] (no total count field)
- POST /api/v1/agents/ body:AgentCreate → Response[AgentRead] (201)
- PATCH /api/v1/agents/{id}/status body:StatusChangeRequest → Response[AgentRead]
- DELETE /api/v1/agents/{id} → Response[None]
- Error responses: 404 (not found), 409 (invalid transition / delete active), 422 (activation not ready)
- Error body: { detail: "具体中文原因" } — displayed by global interceptor

From frontend/src/api/index.js interceptor:
- Returns response.data (the { code, data, message } object) on success
- Calls ElMessage.error() for all errors automatically
- Store actions access payload via res.data (NOT res.data.data)
</interfaces>
