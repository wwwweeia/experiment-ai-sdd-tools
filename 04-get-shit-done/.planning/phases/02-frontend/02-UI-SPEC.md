## Color

Element Plus semantic color palette — no custom theme.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `#f5f7fa` | Page background, table row background, main surface |
| Secondary (30%) | `#f5f7fa` | Search area background, card backgrounds, el-header area |
| Accent (10%) | `#409eff` (EP `--el-color-primary`) | Primary action buttons, active nav menu item, logo text |
| Destructive | `#f56c6c` (EP `--el-color-danger`) | Delete button, destructive confirmation dialog |

**Accent reserved for:**
1. "创建 Agent" primary button (type="primary")
2. "激活" action button for DRAFT agents (type="primary")
3. "激活" action button for INACTIVE agents (type="primary")
4. Active navigation menu highlight
5. Logo text color (`color: #409eff` — matches App.vue)

**Status tag colors (explicit mapping):**

| Status | Element Plus Tag Type | Hex |
|--------|----------------------|-----|
| DRAFT | `type="info"` | `#909399` |
| ACTIVE | `type="success"` | `#67c23a` |
| INACTIVE | `type="warning"` | `#e6a23c` |

> Source: Derived from ROADMAP.md "状态标签（颜色区分三种状态）" requirement. Info/success/warning is the standard EP semantic mapping.

## Interaction Contract

### State Button Rules (per-row)

| Agent Status | Buttons Shown | Button Type |
|-------------|---------------|-------------|
| DRAFT | "激活" | type="primary" |
| ACTIVE | "停用" | type="warning" |
| INACTIVE | "激活" + "删除" | type="primary" + type="danger" |

### Confirmation Dialogs

All state transitions and deletes require `ElMessageBox.confirm()` before API call.

| Action | Confirmation Message |
|--------|---------------------|
| DRAFT → ACTIVE (激活) | 确认激活 Agent「{name}」？激活前系统将验证关联的 Model 和 Prompt 是否有效。 |
| ACTIVE → INACTIVE (停用) | 确认停用 Agent「{name}」？停用后关联的 Skill 将标记为不可用。 |
| INACTIVE → ACTIVE (激活) | 确认重新激活 Agent「{name}」？系统将重新验证关联的 Model 和 Prompt。 |
| INACTIVE → Delete (删除) | 确认删除 Agent「{name}」？此操作不可撤销。 |

### Create Form Validation

| Field | Required | Validation Rule |
|-------|----------|----------------|
| 名称 | Yes | 非空，最大长度 100 字符 |
| 描述 | No | 最大长度 500 字符 |
| Model | Yes (for activation, optional for DRAFT) | 下拉选择；DRAFT 创建时允许为空 |
| Prompt | Yes (for activation, optional for DRAFT) | 下拉选择；DRAFT 创建时允许为空 |

> Note: Model and Prompt are optional at creation time (initial status is always DRAFT). Backend enforces them at activation. UI should allow empty dropdowns at creation.

### Pagination

- Page size: 10 records per page (matches backend default)
- Show total count: "共 {N} 条记录"
- Component: `el-pagination` with `layout="total, prev, pager, next"`

## Implementation Notes

### Follow Existing Patterns (source: codebase scan)

1. **View structure**: Match `PromptList.vue` — `.{name}-list` wrapper, `padding: 20px`, `background: #fff`, `min-height: calc(100vh - 60px)`
2. **Store pattern**: Match `stores/prompt.js` — `defineStore` with Setup Store syntax, `ref()` state, `computed()` getters, async actions with `loading` flag
3. **API client pattern**: Match `api/prompts.js` — flat named exports wrapping `api.get/post/patch/delete`
4. **Error handling**: `ElMessage.error()` is already globally handled in `api/index.js` interceptor — store actions only need `try/catch` for state cleanup
5. **Lazy loading for dropdowns**: Fetch Model and Prompt lists only when the create dialog opens (not on page mount)

### New File Targets

| File | Purpose |
|------|---------|
| `frontend/src/api/agents.js` | Agent API client (fetchAgents, createAgent, updateAgentStatus, deleteAgent, fetchModels, fetchPrompts for dropdowns) |
| `frontend/src/stores/agent.js` | Extend existing stub — add statusFilter, pagination, createAgent, updateStatus, deleteAgent actions |
| `frontend/src/views/AgentList.vue` | Replace 15-line stub with full implementation |
