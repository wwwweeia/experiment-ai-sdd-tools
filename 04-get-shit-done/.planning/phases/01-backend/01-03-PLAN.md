<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → FastAPI router | Untrusted JSON body, query params, path params |
| Router → Service | Validated DTOs cross; service may raise domain exceptions |
| Router → HTTP response | Domain exceptions translated to HTTP status codes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `?status=garbage` query param on GET /agents/ | mitigate | `status: AgentStatus | None = None` typing — FastAPI auto-rejects unknown enum values with 422 before reaching service. |
| T-03-02 | Tampering | POST /agents/ body with extra `status` field | mitigate | AgentCreate (Plan 01) has no `status` field. Pydantic v2 default config silently ignores unknown fields → result is DRAFT regardless of body. |
| T-03-03 | Information Disclosure | Exception `str(e)` leaking stack traces / SQL | mitigate | Service raises plain ValueError subclasses with curated Chinese messages — no exception chaining to inner errors, no `repr(e)`. |
| T-03-04 | Denial of Service | Large `limit` query parameter | accept | `limit: int = 100` default; no upper bound enforced in v1. Acceptable for experiment project; existing `list_models` / `list_prompts` have same exposure. |
| T-03-05 | Spoofing | Wrong HTTP status code for known frontend branch | mitigate | Acceptance criteria explicitly assert `status_code=409` for InvalidTransition and `status_code=422` for ActivationNotReady — Phase 2 frontend depends on this contract. |
</threat_model>
