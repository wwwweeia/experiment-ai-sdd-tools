```python
class AgentStatus(str, enum.Enum):
    DRAFT = "draft"; ACTIVE = "active"; INACTIVE = "inactive"

class Agent(Base):
    id, name, description, model_id, prompt_id, status, created_at
    model: relationship → Model
    prompt: relationship → Prompt
    status_history: list[AgentStatusHistory]

class AgentStatusHistory(Base):
    id, agent_id (FK CASCADE), from_status (nullable enum), to_status (enum),
    reason (nullable str 500), changed_at (datetime, default=datetime.now)
    agent: relationship → Agent

class Model(Base):
    id, name, provider, model_type, version, created_at

class Prompt(Base):
    id, title (NOT name), content, variables, tags, created_by, created_at, updated_at
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service ← Schema | AgentCreate / StatusChangeRequest already validated by Pydantic; service trusts shape but verifies business rules |
| Service → ORM | SQLAlchemy session writes; transaction boundary at `commit()` |
| Service ↔ Database | Cascade delete and FK integrity enforced by ORM + SQLite |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | create_agent honoring an injected status field | mitigate | `Agent(**data.model_dump())` is safe because AgentCreate (Plan 01) has no `status` field. Defense in depth: explicitly set `agent.status = AgentStatus.DRAFT` after construction — even if AgentCreate is later mutated to include status, this overwrite enforces AGENT-03. |
| T-02-02 | Tampering | INACTIVE→ACTIVE re-activation without re-validating model/prompt | mitigate | `_assert_activation_ready()` is called for any `target == AgentStatus.ACTIVE` regardless of `from_status` (Pitfall #6 from RESEARCH.md). DRAFT and INACTIVE share the same activation guard. |
| T-02-03 | Repudiation | Status changed without history row | mitigate | `agent.status = new` and `db.add(AgentStatusHistory(...))` live in the same `db.commit()` transaction. SQLite atomicity guarantees both persist or neither. |
| T-02-04 | Repudiation | Initial DRAFT creation lacks audit trail | mitigate | `create_agent()` writes an initial history row with `from_status=None, to_status=DRAFT, reason="初始创建"`. Provides full audit trail from agent birth. |
| T-02-05 | Denial of Service | N+1 queries on list endpoint with 1000 Agents | mitigate | `selectinload(Agent.model)` + `selectinload(Agent.prompt)` batches relationship loads into 2 IN-clause queries regardless of result count. |
| T-02-06 | Elevation of Privilege | Deleting ACTIVE Agent bypasses status enforcement | mitigate | `delete_agent()` raises `ValueError` before reaching `db.delete()` when `agent.status == AgentStatus.ACTIVE`. Router maps to 409 (Plan 03). |
| T-02-07 | Information Disclosure | Exception messages leaking DB internals | accept | Error messages are Chinese-language business descriptions ("激活失败：关联模型 (id=5) 已不存在"). They expose entity IDs but no schemas/credentials. Acceptable for internal experiment project. |
